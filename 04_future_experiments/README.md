# Future Experiments (Parked)

Work that's been specced or sketched but is not being actively built. Preserved here in case it gets revived later.

## What's here

- **`playbook/`** — full product specification for a trading-journal SaaS called "Playbook enforcement." This was the original direction of the project (April 2026) before it pivoted to a personal site. The spec is thorough and still accurate about the feature it describes. Parked, not deleted, because it might be worth reviving as a site experiment someday.

## What "parked" means

- Not actively being built
- Not on any roadmap
- Not discussed in the site spec
- Available to revisit if/when it makes sense

## When to unpark something

Three signals worth paying attention to:

1. **Personal pull** — a genuine, repeated desire to work on it, not a momentary excitement
2. **Reader/audience pull** — if the site grows and readers start asking for something that matches a parked experiment, that's real signal
3. **Natural fit** — the experiment fits the site's current direction, not an older version of the project

If none of those signals are present, the experiment stays parked. The fact that work has already been done isn't a reason to revive it.

## Adding new parked experiments

If a new idea gets specced but doesn't get built, give it its own folder here alongside `playbook/`. Include a README in that folder explaining what it was and why it's parked. Future-you will appreciate the context.
